# Website Conversion Review

## Use when

Use this skill when a user wants to improve a website page or funnel: landing pages, ecommerce product pages, SaaS signup pages, inquiry pages, service pages, pricing pages, checkout flows, or lead-generation pages.

## Do not use when

Do not use this skill to invent analytics, fabricate user behavior, or claim a conversion lift without A/B test or before/after data.

## Required inputs

- Page URL, screenshots, HTML, copy, or page description.
- Conversion goal: purchase, inquiry, signup, booking, download, subscribe, contact, quote request.
- Target audience.
- Traffic source if known.
- Known analytics or user feedback if available.
- Constraints: brand tone, platform, legal, privacy, performance.

## Evidence basis

- Evidence level: L1/L2. Based on common CRO and UX review practice; L3 requires representative page examples or evals; L4 requires conversion data or A/B tests.
- Underlying logic: Conversion friction usually appears as unclear offer, weak proof, weak CTA, missing answers, excessive form or checkout friction, or trust gaps.
- Source or reference: `docs/skill-source-map.md`; page evidence from screenshots, HTML, copy, analytics, or user feedback.
- Validation method: Compare recommendations against page evidence and verify that metrics are proposed rather than fabricated.
- Update trigger: Review when platform UX constraints, privacy rules, payment/checkout requirements, or target conversion goal changes.

## Workflow

1. Define the conversion goal and expected user journey.
2. Review above-the-fold clarity: what it is, who it is for, why it matters, what to do next.
3. Review offer-message fit: pain point, value proposition, proof, differentiation, urgency.
4. Review trust signals: reviews, certifications, guarantees, policies, security, social proof, company credibility.
5. Review friction: confusing navigation, weak CTA, long forms, missing FAQ, hidden price, slow page, unclear shipping or terms.
6. Review visual hierarchy and section order.
7. Identify missing evidence and risky claims.
8. Recommend changes by impact, effort, and confidence.
9. Define validation: A/B test, event tracking, heatmap, form completion, checkout funnel, inquiry quality.

## Output

Return:

- Conversion goal.
- Page/funnel diagnosis.
- High-impact issues.
- Copy and structure recommendations.
- Trust and proof gaps.
- Friction points.
- Prioritized fix list.
- Validation metrics.

## Validation

A good review ties recommendations to the page’s conversion goal and visible evidence. It does not make performance claims without analytics or tests.

## Safety notes

Do not recommend dark patterns, fake urgency, fake reviews, hidden fees, deceptive claims, or manipulative consent flows. Respect privacy and platform rules.
